Firebase e historico online

Arquivos principais alterados:

- index.html
- style.css
- firebase.js
- script.js
- historico.js
- historicoBobina.js
- historicoAgropainel.js

O projeto ja contem a configuracao Firebase informada no pedido.
Para funcionar online, publique estes arquivos em um servidor HTTPS ou no GitHub Pages e mantenha o Firestore Database ativo no projeto atlas-bobina.

Colecoes usadas no Cloud Firestore:

- usuarios
- calculos_bobina
- calculos_agropainel
- historico_bobina
- historico_agropainel

A colecao historico_bobina registra:

- Data
- Hora
- Tipo da acao
- Usuario
- Produto/Bobina
- Quantidade
- Observacao

A colecao historico_agropainel registra:

- Data
- Hora
- Acao realizada
- Valores antes e depois da alteracao
- Usuario

O historico agora e salvo manualmente pelo botao de cada calculadora.
Na aba Bobina existe o botao "Salvar Bobina no historico".
Na aba Agropainel existe o botao "Salvar Agropainel no historico".

Cada historico local mantem somente os 10 calculos mais recentes.
O sistema tambem tenta gravar os dados no Firebase quando o banco estiver disponivel.

O botao "Testar Banco de Dados" foi removido da tela inicial.
Na aba Bobina aparece somente o Historico Bobina.
Na aba Agropainel aparece somente o Historico Agropainel.
Cada cartao mostra apenas nome do usuario, data e hora, mm da chapa, velocidade e metros.

Na Bobina, o historico so grava depois que o usuario seleciona uma espessura, uma velocidade e clica em salvar.
No Agropainel, o historico so grava depois que o usuario seleciona uma velocidade e clica em salvar.
Calculos repetidos com os mesmos dados nao geram duplicatas.
